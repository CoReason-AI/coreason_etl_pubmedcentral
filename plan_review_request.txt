**Comprehensive Analysis**
The user wants to incrementally build an ETL pipeline for PubMed Central data. The existing codebase implements source fetching (S3/FTP), XML parsing (JATS to epistemic states), surrogate key generation, and basic configuration.

**Decomposition and Planning (Pending Work)**
1. **Manifest Parser (Bronze Layer):** Create a component using DuckDB (`read_csv`) to parse the PMC manifest CSVs, implementing the High-Water Mark (delta) update mechanism based on `last_updated` and extracting the `retracted` status.
2. **Bronze Layer Ingestion Resource:** Create a `dlt` resource that yields records to the `bronze_pmc_file` table containing `source_file_path`, `ingestion_ts`, `ingestion_source`, and `file_metadata`. It must validate that `source_file_path` and `ingestion_source` are strings, and it must stream tarballs to local storage (not storing XML payloads in Postgres).
3. **Silver Layer Transformer:** Create a `dlt` transformer to process files from the Bronze layer using `stream_tarball_xmls`, and apply `generate_surrogate_keys` for `coreason_id`.
4. **Gold Layer Transformer:** Create a `dlt` transformer to denormalize the `ParsedArticleState` into the `gold_pmc_analytics_rich` wide table, fulfilling the dlt schema explosion prevention (either `max_table_nesting=0` on pipeline or JSON dumps).
5. **Main Pipeline Orchestration:** Integrate the sources, transformers, and configuration into `main.py` using `dlt.pipeline()`, and implement the live integration tests.

**Selected Atomic Unit**
**Step 1: Manifest Parser (Bronze Layer)**
- I will implement a new module `src/coreason_etl_pubmedcentral/manifest_parser.py`.
